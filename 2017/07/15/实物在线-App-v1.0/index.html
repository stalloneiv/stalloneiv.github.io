1.3、登录

1.3.1登录

（登录）

字段名称 是否必填 字段长度 初始键盘 提示文字 其他规则
手机号 是 11 数字键盘 请输入手机号码进行登录 第1位必须=1,第二位不能=1/2/6/9,不符合规则提交的时候显示toast”请输入正确的手机号码”
密码 是 6-18 字母全键盘 请输入密码进行登录 输入的内容必须=密码,不符合规则提交的时候显示toast“账号/密码不正确”

（账号、密码规则）

功能描述
返回：返回到上一页
账号/密码录入：规则见上表（账号、密码规则）
注册入口：点击跳转
忘记密码入口：点击跳转
登录提交入口：点击提交系统判断，并显示对应toast
快捷登录入口：点击跳转
录入清空：已录入内容时，显示清空按钮，点击清空输入内容
单点登录：需实现单点登录功能，同一账号同时只能登录一个移动终端APP，具体规则见功能逻辑；
功能逻辑
同一时间同一账户只能登录一个移动终端APP；
用户登录账号时系统需判断是否有其他移动终端(APP)在登录中，如有则强制下线并弹窗提示“您的账号在其他移动终端登录，已下线”；

点击确定关闭弹窗并刷新页面；

1.3.2快捷登录

（社交平台快捷登录）

功能描述
用户点击微信/QQ跳转至微信/QQAPP内获得授权，获得授权后系统需获取用户的微信/QQ昵称+微信/QQ头像，获取完成后跳转至绑定手机页面；
功能逻辑
用户点击微信快捷登录时，系统需判断微信是否绑定手机账号，如已绑定则显示登录成功；如未绑定的情况下则需要跳转至注册页面；

1.4、注册

（新账号注册）

字段名称 是否必填 字段长度 初始键盘 提示文字 其他规则

密码 是 6-18 字母全键盘 请输入密码进行登录 输入的内容必须=规则

密码录入：录入字段
密码提交：提交密码，系统判断，符合规则完成注册流程，不符合显示对应toast，并停留在当前页面
录入清空：已录入内容时，显示清空按钮，点击清空输入内容

1.7.3 共享场景

（共享邀请）

功能描述
微信分享/授权接口：通过接口用户可获得用户微信头像、用户名等信息，并分享邀请卡片至微信好友；
QQ分享/授权接口：通过接口用户可获得用户QQ头像、用户名等信息，并分享邀请卡片至QQ好友；
功能逻辑
./media/image21.png
系统判断已有五个，用户发起第六个时，需有对应文字提示“共享空间创建数量已达上限，请稍后再试”；
当共享空间内无数据传输/用户停留，则即时关闭空间；
关闭空间后用户点击共享链接默认进入普通demo界面；
邀请后即生成共享空间，其他用户通过点击链接即可进入对应页面；
其他用户通过APP分享的链接可进入对应H5页面查看，系统需判断当前机器是否已安装APP，如已安装则需要通过html调起本地APP并进入APP内对应共享空间；
发起共享邀请需要登录；
查看者不需要登录，显示默认头像；

（共享场景-操作者）

功能描述
返回：点击显示弹窗询问“是否退出共享模式？”，点击确定回到上一场景，点击取消关闭弹窗停留当前场景；
音频：操作者音频默认开启，点击可完成关闭/开启操作，显示对应toast“音频已开启/音频已关闭”；
实时语音：打开后用户说话即可传输语音信息，操作者默认开启，点击可完成关闭/开启操作，显示对应toast“实时语音已开启/实时语音已关闭”；
商品信息：场景内显示商品品牌、商品名称；
数据记录：系统需记录用户停留时间，点击按钮的次数与种类；
人员显示：显示当前在线用户社交平台头像，操作者显示操作字段，用户退出后隐藏头像；
功能逻辑
./media/image24.png （退出、关闭空间逻辑）
人数上限：最多同时容纳三名用户访问，满员时有新的用户进入需显示toast“抱歉，共享房间已满员”，并进入非共享的产品使用场景；
房间关闭：系统需判断当前房间是否有用户留存，如判断结果为无，则关闭共享房间；
用户通过外链进入已关闭的房间：默认跳转到产品初始场景；
操作权限更替：操作权限默认在房主（邀请者），房主退出后操作权限按照进入房间的时间进行更替，先到者先得，操作权限仅有一名用户；
操作权限：有操作权限即可对模型进行交互、编辑操作，查看者可实时看到操作动态、效果；
音频：查看者进入共享房间，音频功能默认开启；

2、交互需求

2.1、性能、交互需求

该版本产品需在满足功能需求的前提下尽可能的提高交互体验，交互体验主要体现在以下几点：

  1. APP、U3D产品框架效果需与UI效果图保持一致；

  2. 点击按钮的区域不能出现多次点击但无反应的情况，涉及到提交数据的按钮用户发生点击操作后就需要弹出loding图标，用户不能做重复提交的操作；

  3. 所有没有数据的页面都需要显示缺省页面；

  4. APP内所有需要显示系统提示的功能部分都需按照需求显示对应提示，toast类型的提示显示3秒后需自动消失；

  5. 页面切换需快速流畅；

  6. 产品运行时需稳定流畅；

  7. 并发需求：

  • 系统需支持同时做不同操作的用户数量为：总用户数量×5%~20%

系统响应的平均时间为5s内

  • 系统需支持同时做一样操作的用户数量为：300人

系统响应的平均时间为5s内

3、安全要求

在实物在线APPv1.0版本中，对安全要求的内容为：

  1. 涉及到U3D部分内容要求不能反编译；

  2. 涉及到用户的账户、密码、支付密码的相关内容都需加密传输存储；

  3. 涉及到支付流程、订单相关的内容在传输时需加密传输；

  4. 系统不能有安全方面的漏洞；

4、硬件环境

安卓系统：要求软件在安卓系统4.4版本以上，包含4.4版本的安卓手机上运行流畅，无卡顿闪退情况。

IOS系统：要求软件在IOS系统8.0版本以上，包含8.0版本的苹果手机上运行流畅，无卡顿闪退情况。
